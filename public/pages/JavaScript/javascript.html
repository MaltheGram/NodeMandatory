Variables in JavaScript

A variable is a “named storage” for data. We can use variables to store goodies, visitors, and other data. To create a variable in JavaScript, we use the let or const keyword. The statement below declares a variable with the name “message”: const message = "Hello World" Constants are block-scoped, much like variables declared using the let keyword. The value of a constant can't be changed through reassignment (i.e. by using the assignment operator), and it can't be redeclared (i.e. through a variable declaration). However, if a constant is an object or array its properties or items can be updated or removed.

Callback Functions

A callback function is a function passed into another function as an argument, which is then invoked inside the outer function to complete some kind of routine or action.

Arrow Functions

Arrow functions is another way to make a function. It can be more compact const myArrowFunction = (a, b) => a + b
vs
function myNormalFunction(a, b) { return a + b }
  • Arrow functions don't have their own bindings to this, arguments or super, and should not be used as methods.
  • Arrow functions don't have access to the new.target keyword.
  • Arrow functions aren't suitable for call, apply and bind methods, which generally rely on establishing a scope.
  • Arrow functions cannot be used as constructors.
  • Arrow functions cannot use yield, within its body.
Source: Developer Mozilla

Type Coercion

In JavaScript when we want to compare two values, we will use === rather than ==. 
                
                    const price = 12
                    const priceString = "12"
                    const concat = price + priceString

                    result: "1212"
You will not necessarily compare a string to a number, and expect this result. So if we wanted compare these two values, there's one way we would NOT do it, and one proper way. 
                
                    const price = 12
                    const priceString = "12"
                    const isSame = price == priceString

                    result: true

                    const isSame = price === priceString

                    result: false
Moral of the story: Always use === to compare variables and be aware of type coercion.
